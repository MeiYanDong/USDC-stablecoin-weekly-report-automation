必须生成的文件与内容：

1) stablecoin_weekly.py
功能要求：
- Python 3.11
- 从 DefiLlama 拉取稳定币供给数据，计算三项
  a total_supply_usd：全市场稳定币总供给，仅 peggedUSD，取最新日期
  b usdc_supply_usd：USDC 供给，仅 peggedUSD，取最新值
  c usdc_supply_share：usdc_supply_usd / total_supply_usd
- 从 Dune 拉取稳定币 7 天链上转账量结果，计算一项
  d usdc_transfer_volume_share_7d：过去 7 天 USDC 的链上转账量 / 分母集合过去 7 天链上转账量之和
- 分母集合定义：从 DefiLlama 的稳定币列表中过滤 pegType 为 peggedUSD，按 circulating 降序取 top20 的 symbol。将 symbol 统一转为小写。若 Dune query 结果缺失某 symbol，跳过并加入 missing_symbols 列表，同时分母里不计入该 symbol。
- 统计区间定义：以 Asia/Shanghai 的日期为准，end_date 取昨天，start_date 取 end_date 往前 6 天，一共 7 天。Dune query 直接在 SQL 内按北京时间计算该窗口，不依赖手工传参。
- 可靠性：使用 requests，请求超时 20 秒；使用 tenacity 做指数退避重试；任何异常要捕获，仍然要给飞书发一条失败通知，内容包含失败原因和已成功拿到的部分指标。
- 输出：推送飞书一条消息卡片，信息包含
  1 本次统计日期，北京时间
  2 四个指标的数值，供给金额单位 USD，保留 2 位小数，份额百分比保留 2 位
  3 与上一次周报相比的 WoW 变化，若没有上次数据则显示 N/A
  4 missing_symbols 列表，若为空显示 none
  5 数据源说明，Supply 来自 DefiLlama，Transfer Volume 来自 Dune
- 落地保存：把每次周报结果追加保存到 data/weekly_history.json，至少保留最近 52 条。结构建议为数组，每条包含 run_time_beijing, end_date, start_date, metrics, missing_symbols。
- 时区门禁：脚本启动时检查当前 Asia/Shanghai 时间，只有周一 07:00 才继续执行。若不是该时间点，直接正常退出返回码 0。为了方便手动调试，允许设置环境变量 FORCE_RUN=1 时跳过门禁。
- 配置全部走环境变量：
  DUNE_API_KEY
  DUNE_QUERY_ID
  FEISHU_WEBHOOK_URL
  FORCE_RUN 可选
- 飞书发送：使用群聊自定义机器人 webhook，走 POST JSON。卡片格式用飞书交互式消息卡片即可，失败时退化为纯文本也可以。

接口要求：
- DefiLlama
  GET https://stablecoins.llama.fi/stablecoins
  用于拿 stablecoin 列表，字段包含 symbol, pegType, circulating
  GET https://stablecoins.llama.fi/stablecoincharts/all
  用于拿全市场总量时间序列，取最后一天 totalCirculating.peggedUSD
- Dune
  使用 Query Execute + Execution Status + Execution Results API 获取 query 输出结果
  query 至少输出 symbol, volume_7d_usd 两列
  query 内部按北京时间自行计算过去 7 天、截止昨天的统计窗口
